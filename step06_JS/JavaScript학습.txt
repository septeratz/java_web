4-1. 변수
     var / const / let 키워드
     선언 위치에 따라 로컬변수, 전역변수로 구분

     1. 선언 방법
        - type 선언은 없음, type은 있음
        - 동적으로 데이터 대입시 자동으로 유동적으로 타입 결정
        변수명
            - 선언 위치가 함수 내부라 하더라도 전역 변수로 간주함
            - 권장하지 않음
        var 변수명
            - 함수 내부에 선언 시 해당 함수의 모든 영역에서 호출 가능
            - js 문법 생성 시점에 제시된 키워드
        let 변수명
            - 함수에 선언 시 주의사항
                : if문과 같이 중첩된 중괄호 내에 선언 시 중괄호 내에서만 사용 가능
                : 사용 범위의 명확성 부여
                : js가 버전업되면서 새로 추가된 스펙
                : var 대신 let 사용 적극 권장
        const 변수명
            - 상수, 값 변경 불가
            - client에게 빠른 응답을 위한 실행 속도 고려시 const 키워드 사용 권장
                : 값 불변 변수에 한해서

5-2. XMLHttpRequest
    - 기원
         : xml이 데이터 포멧으로 사용되던 시절에 제시된 이름
         : 웹사이트가 동적이지 못하던 시절
         : 서버가 client에게 데이터를 전송하던 주 포맷
            일반 text, csv, xml
            xml의 단점: 데이터 양이 많을수록 tag로 필요 데이터 착출 및 활용
                실행속도 이슈
            해결책: key로 value 구분해서 가볍게 처리 가능한 포맷 JSON
                JavaScript Object Notation
                - 현 시점에선 데이터 포맷의 으뜸, 다양한 설정으로 사용되는 중
    - 참고
        MS만 ActiveObject로 밀어붙임, 그러다 현재는 XMLHttpRequest 객체로 작업 같이 권장
    
    - 🌟 암기 사항
        - html 문서 상에서 페이지 이동 또는 refresh 없이 서버에 요청
        - 요청으로 인한 응답도 받음
            - 하나의 객체가 요청 및 응답 수행
    
    - 필요 기능
        1. http 통신 규약 인지(get, post, ..)
        2. 어떤 서버의 어떤 프로그램에 요청할지 정보
        3. 요청중? 응답중? 파악 필요
        4. 응답 완료시 정상응답(200)? 비정상 응답(404, 405, 500..)?
